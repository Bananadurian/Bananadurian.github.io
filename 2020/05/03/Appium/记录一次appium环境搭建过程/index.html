记录一下Appium+Python环境搭建过程

需要的环境

1、JDK环境

JAVA的开发环境JDK，我用的是1.8.0版本

2、Android SKD

Google提供的Android开发工具包，开发Android应用时需要通过引入其工具包来使用Android相关的API

3、Node.js

Node.js就是运行在服务端的 JavaScript，appium服务端是node.js写的，所以安装那个平台都是先安装Node.js

npm（node package manager）命令可以安装appium，npm是Node.js的包管理工具

4、Appium desktop

5、Appium-doctor

appium-doctor可以检测Appium整体依赖环境配置情况。

6、Appium-Python-Client

python依赖库

详细安装过程

1、JDK安装

此次安装的是1.8.0版本 官网下载地址

……但是由于官网需要注册，由于注册过程繁琐，最后随便找了一个别人下载好的安装包。

  • 打开下载好的jdk-8u231-windows-x64.exe，一直按照提示点击下一步即可（中间可根据自己的情况安装开始的时候选择jdk的安装路径，安装过程中选择jre的安装路径）

  • 设置环境变量

i、新建系统变量 Java_Home

变量名： Java_Home 变量值：JDK的安装路径，我的路径为“C:\Tool_Software\JDK8”

ii、新建系统变量 CLASSPATH

变量名：CLASSPATH

变量值：.;%Java_Home%\bin;%Java_Home%\lib\dt.jar;%Java_Home%\lib\tools.jar

iii、编辑系统变量中的PATH变量

直接在原有的PATH变量值中加入：%Java_Home%\bin;%Java_Home%\jre\bin

特别需要注意变量路径之间存在一个;分割。

iv、验证环境是否安装成功

分别在CMD中输入：java；javac；java -version，若没有报错，或提示找不到命令，即说明Java环境已经安装成功。

2、Android SDK安装

2.1 下载sdk安装命令工具

打开官网，网页下拉到底部的Command line tools only，选择对应的SDK tools package下载即可。

官网下载地址

2.2 安装对应版本的SDK

下载完成后，解压文件，在tools目录下的bin目录中，打开CMD，输入下方命令即可开始安装下载最新的platfrom-tools，以及对应版本的platforms和build-tools，安装过程中，会弹出提示框Accept?(y/N) 一直yes即可。

注：上方命令下载的是Android 9相关内容，更多内容以及其它版本的选择参考sdkmanager命令详解

安装成功之后会在tools的上级目录中出现对应的内容，包括但不限于build-tools、emulator、licenses、patcher、platforms、platform-tools、tools等目录。

platform-tools：存放的是pc端和移动端进行交互的工具：比如adb工具，sqlite3工具

build-tools：android打包构建工具

tools：存放一些pc平台下使用的工具：比如模拟器，.9patch工具，uiautomatorviewer可视化工具。

platforms：存放的是api版本

若执行sdkmanager命令的时候报下方错误：

解决方法：

解压出来中的tools改名为latest，tools的 上级目录 改名为cmdline-tools。即\cmdline-tools\latest，此时再进入latest中的bin目录中，打开CMD执行sdkmanager命令即可

参考链接

3.3 配置环境变量

新建系统变量ANDROID_HOME

变量名： ANDROID_HOME 变量值：Android SKD的根目录，我的路径为”C:\Tool_Software\AndroidSDK“

编辑系统变量中的PATH变量

直接在原有的PATH变量值中加入platfrom-tools和tools的地址

验证环境变量是否生效

运行cmd打开命令提示窗，输入where adb 能够显示出其所在路径即OK。

查看Android_home的环境变量是否正确，可以在tools/bin目录下输入echo %android_home% 能够显示出android sdk安装路径即OK。

或者直接输入adb和Android -h若返回一堆信息，也可说明环境安装成功了。

3、Node.js安装

直接在官网下载对应的安装包安装即可。安装成功之后，输入node -v，若返回版本号，即代表安装成功。

4、Appium desktop 安装

为什么是Appium desktop而不是Appium server？

Appium server大约两年前就不在更新了（可以正常使用，没有任何影响）。

Appium-desktop工具其实也封装了Appium server和Node.js依赖环境

直接在官网下载对应Appium desktop安装包后，直接安装即可。

5、Appium-doctor（选择性安装）

由于最新版的Appium desktop是不带 appium-doctor的，所以还需要自己安装 appium-doctor，命令如下：

使用npm install -g appium-doctor将安装在全局环境中。

若要安装在当前目录命令：npm install appium-doctor。安装完后，需要将appium-doctor的路径添加到环境变量里。在当前用户下安装的路径一般是：C:\Users\username\node_modules.bin

输入：“appium-doctor –version” 即可检查appium-doctor是否安装成功

安装成功之后在cmd输入appium-doctor即可检查整体环境是否配置正确

6、安装Appium-Python-Client

若安装很慢，可以尝试使用阿里等pip源

pip install Appium-Python-Client -i http://mirrors.aliyun.com/pypi/simple –trusted-host mirrors.aliyun.com

到这里就安装完成啦！！！
